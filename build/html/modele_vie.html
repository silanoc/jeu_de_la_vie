modele_vie

Dans une achitecture MVC, la partie modele du jeu de la vie. Contient la class Automate, qui est le moteur du jeu.

class jeu_de_la_vie.modele_vie.Automate(taille=(10, 10))

Objet de base du jeu. Basé autour d’une grille (liste de liste). Posséde les méthodes pour se remplir, regarder l’état des cellules voisines et générer le nouvel état.

initialise le jeu avec une grille de 10 sur 10 par défaut. Toutes les cellules sont initialisé avec « « . Permet d’avoir self.grille d’utilisable comme matrice pour tout le reste.

Paramètres

taille (turple) – définit le nombre de lignes et de colonnes voulues. Par defaut 10 sur 10 (10, 10)

calcul_etat_initial()

Pour chaque cellule de self.grille compte le nombre de voisines vivantes.

Incrémente dans le tableau grille_de_calcul la valeur de l’état de chacune des voisine de self.grille.

Renvoie

grille_de_calcul (liste de liste), valeurs allant de 0 à 8.

Type renvoyé

list

donne_etat_final(valeurs_des_voisins)

pour chaque cellule de self.grille defini sont nouvel état (0 ou 1).

Applique les régles du jeu écrites dans les deux dictionnaires. Cela permet de simplifier le style et changer les régles si on veut.

Ecrit directement dans la self.grille le nouvel état.

Paramètres

valeurs_des_voisines (list) – (liste_de_liste), valeurs entre 0 et 8, calculées dans calcul_etat_initial().

genere_nouvelle_generation()

Calcul d’une génération du jeu pour toutes les cellules. Nécessaire pour afficher dans la console.

Enchaine les deux méthodes afin de calculer le nouvel état de chaque cellule de self.grille

remplissage_aleatoire_a_la_creation()

créé et retourne une grille de la même taille que celle initiale, avec des 0 et 1 aléatoire dedans.

Renvoie

remplissage (liste de liste) dont toutes les valeurs sont des 0 ou des 1.

Type renvoyé

list
